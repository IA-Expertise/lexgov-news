## Roadmap discutido (prompt Gemini + prioridades combinadas)

1. **Crítico — ambiente Gemini:** uma variável canônica + fallback + `.env.example` + mensagem de erro coerente.
2. **Slogan:** consistência do texto de marca (e política para `metadata`).
3. **Storage:** interface `StorageProvider`; URLs no banco como path local **ou** URL externa (R2/S3); assinatura de URL = fase posterior se necessário.
4. **Ingestão — categorização:** Gemini Flash com **fallback** para heurísticas e/ou flag; avaliar custo por item e timeout.
5. **White-label voz:** garantir 100% das chamadas TTS com `voiceId` do tenant (já parcialmente ok).
6. **A11y:** Orb, Captions, ARIA, contraste, teclado; banner sugestivo se não for Chrome/Edge.
7. **Admin `/admin/[city]`:** listagem + status áudio + regerar TTS — **exige auth** (definir estratégia: demo vs produção).
8. **Saudação dinâmica:** bom dia/boa tarde + nome da cidade — decidir se MP3 na ingestão (fica “fixo”) vs parte em TTS no cliente.
